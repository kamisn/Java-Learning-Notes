多线程
    Java语言内置了多线程支持：
    一个Java程序实际上是一个JVM进程，JVM进程用一个主线程来执行main()方法，
    在main()方法内部，我们又可以启动多个线程。此外，JVM还有负责垃圾回收的其他工作线程等。
    Java多线程编程的特点又在于：
    因此，对于大多数Java程序来说，我们说多任务，实际上是说如何使用多线程实现多任务。
    和单线程相比，多线程编程的特点在于：多线程经常需要读写共享数据， 并且需要同步。
    例如，播放电影时，就必须由一个线程播放视频，另一个线程播放音频，两个线程需要协调运行，否则画面和声音就不同步。因此，多线程编程的复杂度高，调试更困难。
        多线程模型是Java程序最基本的并发模型；


线程池
在 Java 中，我们可以把一段业务代码包装成 Runnable 或 Callable 任务，然后通过 execute() 或 submit() 提交给线程池执行。
Lambda 表达式本质上就是 Runnable 或 Callable 的简写形式。

ThreadPoolExecutor 的核心构造参数有：

corePoolSize：核心线程数
maximumPoolSize：最大线程数
keepAliveTime：非核心线程空闲存活时间
unit：时间单位
workQueue：任务等待队列
threadFactory：线程工厂
handler：拒绝策略

线程池处理任务的基本流程是：

如果当前线程数小于 corePoolSize，就创建核心线程执行任务。
如果核心线程满了，就尝试把任务放入 workQueue。
如果队列满了，再判断当前线程数是否小于 maximumPoolSize，如果小于，就创建非核心线程执行任务。
如果队列也满了，线程数也到达 maximumPoolSize，就触发拒绝策略。

空闲线程会从任务队列中取任务执行。
对于常见 FIFO 队列，通常会先取等待更久的任务，但线程池整体不保证严格全局有序，因为新任务可能通过扩容线程直接执行。

常见拒绝策略有四种：

AbortPolicy：默认策略，直接抛异常
CallerRunsPolicy：由提交任务的线程自己执行任务，不抛异常
DiscardPolicy：直接丢弃任务
DiscardOldestPolicy：丢弃队列中最老的任务，再尝试提交当前任务

不同队列会影响线程池行为：

ArrayBlockingQueue：有界队列，先排队，队列满了才扩线程
LinkedBlockingQueue：无界队列，任务大多进入队列，通常很难触发非核心线程扩容
SynchronousQueue：不存任务，任务来了通常直接促使线程池创建线程，直到达到最大线程数
