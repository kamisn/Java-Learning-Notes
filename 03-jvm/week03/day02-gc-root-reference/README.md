# JVM 今日复习笔记（4.14）

## 一、JVM 内存结构

### 知识点

#### 1. 线程私有区域

* 程序计数器
* 虚拟机栈
* 本地方法栈

#### 2. 线程共享区域

* 堆
* 方法区

#### 3. 各区域作用

* **程序计数器**：记录当前线程下一条要执行的字节码指令地址
* **虚拟机栈**：存放方法调用相关的信息，如局部变量、操作数栈等
* **本地方法栈**：服务于 Native 方法
* **堆**：存放对象实例，是 GC 的主要回收区域
* **方法区**：存放类信息、常量、静态变量等内容

---

### 面试问答

#### Q1：JVM 内存结构中，哪些区域是线程私有的，哪些是线程共享的？

**答：**
线程私有的有程序计数器、虚拟机栈和本地方法栈；线程共享的有堆和方法区。

#### Q2：为什么 GC 主要回收堆，而不是栈？

**答：**
因为栈帧会随着方法执行结束自动出栈，内存释放比较确定；而堆中的对象不会随着方法结束自动释放，可能被多个地方引用，所以需要 GC 来管理其生命周期。

---

## 二、对象创建流程

### 知识点

#### 对象创建的 5 个步骤

1. **类加载检查**
   检查要创建对象所属的类是否已经加载

2. **分配内存**
   在堆中为对象分配空间

3. **零值初始化**
   将对象实例字段设置为默认零值

4. **设置对象头**
   对象头主要包括：

    * Mark Word
    * 类型指针

5. **执行 `<init>` 方法**
   执行构造器中的初始化逻辑，完成程序员定义的初始化

---

### 面试问答

#### Q1：对象从 `new` 到真正可用，大致经历哪些步骤？

**答：**
首先检查类是否已经加载；如果没有，就先完成类加载。然后在堆中为对象分配内存，再把实例字段初始化为零值默认值，接着设置对象头，最后执行 `<init>` 方法，完成构造器中的初始化逻辑。

#### Q2：执行 `<init>` 之前，对象已经能用了吗？

**答：**
对象在执行 `<init>` 之前已经分配了内存，也有对象头和默认值，但程序员在构造器中定义的初始化逻辑还没执行完，所以不能算完全按预期初始化完成。

#### Q3：对象头里有什么？

**答：**
对象头中主要有 Mark Word 和类型指针。Mark Word 里会记录哈希信息、锁状态、GC 年龄等内容，类型指针用于标识这个对象属于哪个类。

## 三、GC 基础

### 知识点

#### 1. GC 是什么

GC，`Garbage Collection`，即垃圾回收机制。
作用是回收堆中不再使用的对象所占用的内存。

#### 2. 为什么需要 GC

* Java 程序会不断创建对象
* 对象主要存放在堆中
* 如果无用对象不被清理，会长期占用内存
* 最终可能导致内存不足甚至 OOM

#### 3. GC 回收的核心对象

* 主要回收的是 **堆中的垃圾对象**
* 栈不是 GC 的主要回收区域

---

### 面试问答

#### Q1：GC 为什么存在？

**答：**
因为 Java 程序运行过程中会不断创建对象，而这些对象主要存放在堆中。如果无用对象不及时清理，就会一直占用内存，最终可能导致内存不足。GC 的作用就是自动回收这些不再使用的对象。

#### Q2：GC 主要回收哪里？

**答：**
GC 主要回收堆中的对象。因为堆中的对象不会像栈帧那样随着方法结束自动释放，所以需要 GC 专门管理。

---

## 四、对象存活判定：可达性分析

### 知识点

#### 1. JVM 如何判断对象是否存活

主流 JVM 使用的是 **可达性分析**，而不是引用计数法。

#### 2. 可达性分析的核心逻辑

* 从 **GC Roots** 出发
* 沿着 **引用链** 向下搜索
* 能找到的对象：存活
* 找不到的对象：垃圾对象

#### 3. 常见 GC Roots

* 虚拟机栈中引用的对象
* 方法区中类静态变量引用的对象
* 本地方法栈中 JNI 引用的对象

---

### 面试问答

#### Q1：JVM 怎么判断对象该不该回收？

**答：**
JVM 主要通过可达性分析来判断对象是否存活。它会从 GC Roots 出发，沿引用链向下搜索。能够被找到的对象说明还存活，找不到的对象会被视为垃圾对象。

#### Q2：为什么 Java 不主要使用引用计数法？

**答：**
因为引用计数法无法很好解决循环引用问题。两个对象即使已经无法被外部访问，但如果它们互相引用，引用计数仍然不为 0，就可能误判为存活对象。

#### Q3：什么是 GC Roots？

**答：**
GC Roots 可以理解为可达性分析的起点。JVM 会从这些根对象出发，检查其他对象是否还能被访问到。

---

## 五、分代收集思想

### 知识点

#### 1. 为什么要分代

因为不同对象的生命周期不同：

* 大多数对象朝生夕死
* 少数对象会长期存活

#### 2. 分代的核心目标

把不同生命周期的对象分开管理，采用不同的回收策略，提高 GC 效率。

#### 3. 分代结果

* **年轻代**：新创建、短命对象
* **老年代**：经历多次 GC 后仍然存活的对象

---

### 面试问答

#### Q1：JVM 为什么采用分代思想？

**答：**
因为对象的生命周期不同。大多数对象创建后很快死亡，少数对象会长期存活。JVM 把它们分成年轻代和老年代，并针对不同代采用不同回收策略，以提高回收效率。

#### Q2：年轻代和老年代最本质的区别是什么？

**答：**
最本质的区别是对象生命周期不同。年轻代主要存放短命对象，老年代主要存放存活时间更长的对象。

---

## 六、年轻代结构与对象流转

### 知识点

#### 1. 年轻代的组成

* Eden 区
* Survivor From 区
* Survivor To 区

也常记作：

* Eden
* S0
* S1

#### 2. 对象进入路径

* 新对象一般先进入 **Eden**
* Eden 满时触发 **Minor GC**
* 存活对象复制到 **Survivor**
* 之后在两个 Survivor 区之间轮换复制
* 年龄达到阈值后晋升到 **老年代**

#### 3. Minor GC 的特点

* 主要回收年轻代
* 年轻代对象存活率通常较低
* 因此复制成本相对较低

---

### 面试问答

#### Q1：新创建的对象一般先进入哪里？

**答：**
一般先进入 Eden 区。

#### Q2：Minor GC 主要回收哪一部分？

**答：**
Minor GC 主要回收年轻代。

#### Q3：Minor GC 的典型触发点是什么？

**答：**
典型触发点是 Eden 区空间不足或接近满了。

#### Q4：为什么年轻代要有两个 Survivor 区？

**答：**
因为年轻代通常采用复制算法，两个 Survivor 区可以轮换扮演 From 和 To 的角色，用于存放 GC 后仍然存活的对象。

#### Q5：对象什么时候进入老年代？

**答：**
对象在经历多次 Minor GC 后如果仍然存活，并且年龄达到阈值，就会晋升到老年代。

---

## 七、老年代回收特点

### 知识点

#### 1. 老年代对象特点

* 存活率更高
* 回收频率相对较低
* 回收成本更高

#### 2. 常见处理思路

* 标记-清除
* 标记-整理

#### 3. 标记-整理法

* 先标记存活对象
* 把存活对象移动到内存一端
* 清理边界外空间
* 减少内存碎片

---

### 面试问答

#### Q1：为什么老年代通常不用复制算法作为主要思路？

**答：**
因为老年代对象存活率更高，如果采用复制算法，需要复制的对象会很多，成本较高，所以更常采用标记-清除或标记-整理思路。

#### Q2：标记-整理法的优点是什么？

**答：**
它可以减少内存碎片，使后续分配对象时更方便。

---

## 八、G1 垃圾收集器

### 知识点

#### 1. G1 是什么

G1，`Garbage First`，是一种具体的垃圾收集器。

#### 2. G1 的核心特点

* 仍然保留分代思想
* 把整个堆划分成多个 **Region**
* 不同 Region 可以按需扮演 Eden、Survivor、Old 等角色
* 优先回收垃圾多、收益高的 Region

#### 3. G1 的优势

* 回收范围更灵活
* 停顿时间更可控
* 更适合大堆场景

---

### 面试问答

#### Q1：G1 的核心特点是什么？

**答：**
G1 仍然保留分代思想，但它不会把堆简单划成连续的一整块年轻代和老年代，而是把整个堆划分成多个 Region，并按需让这些 Region 扮演不同的角色。它会优先回收垃圾较多、收益更高的 Region。

#### Q2：为什么叫 Garbage First？

**答：**
因为 G1 会优先选择垃圾更多、回收收益更高的 Region 来进行回收，所以叫 Garbage First。

#### Q3：G1 的优势是什么？

**答：**
G1 的优势是回收更灵活，停顿时间更容易控制，因此比较适合大堆以及对响应时间有要求的场景。

#### Q4：G1 和经典分代收集器相比，有什么区别？

**答：**
G1 仍然有分代思想，但它通过 Region 化管理堆内存，不再依赖传统那种更固定的大块分区回收方式，因此在回收范围选择上更灵活，停顿时间也更可控。

## Q1：请你从 JVM 内存结构讲到对象创建，再讲到 GC。

**答：**
JVM 内存结构可以分为线程私有和线程共享两部分。线程私有的主要有程序计数器、虚拟机栈和本地方法栈；线程共享的主要有堆和方法区。对象一般分配在堆中。

## Q2：GC 是怎么工作的？

**答：**
GC 的作用是回收堆中不再使用的对象。JVM 通过可达性分析，从 GC Roots 出发判断对象是否还可达。为了提高回收效率，JVM 采用分代收集思想，把对象分成年轻代和老年代。新对象一般先进入 Eden，Eden 满了触发 Minor GC，存活对象进入 Survivor，并在多次 GC 后晋升到老年代。具体收集器中，G1 会把堆划分成多个 Region，并优先回收垃圾较多的区域，使停顿时间更可控。

class Solution {
List<List<Integer>> result = new ArrayList<>();
LinkedList<Integer> path = new LinkedList<>();

    public List<List<Integer>> combine(int n, int k) {
        backtracking(n, k, 1);
        return result;
    }

    public void backtracking(int n, int k, int startIndex) {
        if (path.size() == k) {
            result.add(new ArrayList<>(path));
            return;
        }

        for (int i = startIndex; i <= n; i++) {
            path.add(i);
            backtracking(n, k, i + 1);
            path.removeLast();
        }
    }
}
