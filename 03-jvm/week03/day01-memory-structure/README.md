## 一、JVM：运行时数据区

### 知识点 1：JVM 运行时数据区的整体划分

JVM 的运行时数据区可以分为两大类：

* **线程私有**

    * 程序计数器
    * 虚拟机栈
    * 本地方法栈
* **线程共享**

    * 堆
    * 方法区

补充：

* **运行时常量池** 属于 **方法区的一部分**
* **元空间** 是 JDK 8 之后 HotSpot 对 **方法区** 的实现，使用的是 **本地内存**

#### 面试问答

**Q：请你说一下 JVM 运行时数据区是怎么划分的？**

**A：**
JVM 的运行时数据区可以分为线程私有和线程共享两部分。
线程私有的有程序计数器、虚拟机栈和本地方法栈，它们主要保存当前线程自己的执行状态。
线程共享的有堆和方法区，其中运行时常量池属于方法区的一部分。
JDK 8 之后，HotSpot 使用元空间来实现方法区，元空间使用的是本地内存。

---

### 知识点 2：为什么要区分线程私有和线程共享

这样划分是为了同时满足两件事：

* **保证线程执行上下文独立**
* **保证多个线程可以共享对象和类信息**

#### 面试问答

**Q：为什么 JVM 要划分线程私有和线程共享区域？**

**A：**
因为 JVM 既要保证每个线程执行时有独立的运行现场，又要支持多个线程共享对象和类信息。
线程私有区域主要保存当前线程自己的执行状态，比如方法调用过程和执行位置。
线程共享区域主要保存多个线程可能共同访问的数据，比如对象实例、类信息和静态变量。

---

## 二、JVM：程序计数器、栈、堆、方法区

### 知识点 3：程序计数器

程序计数器可以理解为：

* 当前线程执行字节码的“书签”
* 记录下一条要执行的字节码指令地址

特点：

* 线程私有
* 不会发生 OOM

#### 面试问答

**Q：程序计数器是什么？有什么作用？**

**A：**
程序计数器记录的是当前线程下一条要执行的字节码指令地址，可以理解为线程执行代码时的书签。
线程切换后，线程能够根据程序计数器恢复到原来的执行位置继续运行。

**Q：为什么程序计数器不会发生 OOM？**

**A：**
因为 Java 虚拟机规范明确规定，程序计数器是唯一一个不会发生 OutOfMemoryError 的区域。

---

### 知识点 4：虚拟机栈与栈帧

虚拟机栈描述 Java 方法执行的内存模型。

特点：

* 每调用一个方法，就会创建一个对应的 **栈帧**
* 方法执行结束后，栈帧出栈
* 当前执行的方法对应的栈帧一定在栈顶

栈帧主要包含：

* 局部变量表
* 操作数栈
* 动态链接
* 方法返回地址

#### 面试问答

**Q：什么是虚拟机栈？**

**A：**
虚拟机栈描述的是 Java 方法执行的内存模型。
每调用一个方法，JVM 就会为这个方法创建一个栈帧并压入栈中；方法执行结束后，栈帧出栈。
当前正在执行的方法一定对应栈顶栈帧。

**Q：栈帧里主要有什么？**

**A：**
栈帧里主要包括局部变量表、操作数栈、动态链接和方法返回地址。
局部变量表用来存基本类型变量和对象引用，操作数栈用于方法执行过程中的运算。

**Q：局部变量表里存的是对象本身还是对象引用？**

**A：**
局部变量表里存的是对象引用，不是对象本身。
因为对象本身通常分配在堆中，栈帧里只需要保存访问这个对象的引用即可。

**Q：一个方法执行结束后，对应的栈帧会怎样？**

**A：**
方法执行结束后，对应的栈帧会出栈，控制权回到上一个方法对应的栈帧。

---

### 知识点 5：本地方法栈

本地方法栈和虚拟机栈类似，但它服务的是：

* **native 方法**

#### 面试问答

**Q：本地方法栈和虚拟机栈有什么区别？**

**A：**
虚拟机栈服务的是 Java 方法调用，本地方法栈服务的是 native 方法调用。
两者本质上都和方法执行有关，只是服务对象不同。

---

### 知识点 6：堆

堆的主要作用：

* 存放对象实例
* 存放数组

特点：

* 线程共享
* 可能被多个线程同时访问
* 是垃圾回收重点区域

#### 面试问答

**Q：堆中主要放什么？**

**A：**
堆中主要存放对象实例和数组。

**Q：为什么堆是线程共享区域？**

**A：**
因为堆中存放的是对象实例和数组，而同一个对象可能被多个线程同时访问，所以堆必须设计成线程共享区域。

---

### 知识点 7：方法区、运行时常量池、元空间

方法区主要存放：

* 类信息
* 静态变量
* 常量
* 即时编译后的代码缓存等

补充关系：

* **运行时常量池** 属于 **方法区**
* **元空间** 是 JDK 8 之后 HotSpot 对 **方法区** 的实现

#### 面试问答

**Q：方法区和堆的区别是什么？**

**A：**
堆主要存放对象实例和数组。
方法区主要存放类信息、静态变量、常量以及即时编译后的代码缓存等内容。
简单来说，堆主要放对象，方法区主要放类相关信息。

**Q：JDK 8 之后方法区发生了什么变化？**

**A：**
JDK 8 之前，HotSpot 用永久代来实现方法区。
JDK 8 之后，取消了永久代，改用元空间来实现方法区。
元空间使用的是本地内存，因此扩展性更好。

**Q：方法区、运行时常量池、元空间三者是什么关系？**

**A：**
方法区是逻辑概念，运行时常量池属于方法区的一部分。
元空间是 JDK 8 之后 HotSpot 对方法区的实现方式。

---

## 三、JVM：对象创建流程

### 知识点 8：对象创建的 5 个步骤

执行 `new Object()` 时，JVM 大致经历以下步骤：

1. 类加载检查
2. 分配内存
3. 初始化零值
4. 设置对象头
5. 执行 `<init>` 构造器

#### 面试问答

**Q：请你说一下 `new Object()` 底层发生了什么？**

**A：**
当执行 `new Object()` 时，JVM 首先会检查 `Object` 类是否已经被加载、解析和初始化，如果没有，就先完成类加载过程。
然后在堆中为对象分配内存。
分配完成后，会先对对象占用的内存进行零值初始化。
接着设置对象头，记录对象运行时需要的信息。
最后执行 `<init>` 构造器，完成程序员定义的初始化逻辑，对象才算真正创建完成。

---

### 知识点 9：类加载检查

对象创建的第一步不是检查“有没有相同对象”，而是：

* 检查这个类能不能被拿来创建实例
* 即检查类是否已经被加载、解析、初始化

#### 面试问答

**Q：对象创建时为什么第一步要进行类加载检查？**

**A：**
因为 JVM 在创建对象之前，必须先确认这个类是否已经被加载、解析和初始化。
如果类本身都还没有准备好，就不能直接创建它的实例。

---

### 知识点 10：对象内存分配方式

对象分配内存有两种常见方式：

* **指针碰撞**

    * 适用于内存规整的情况
* **空闲列表**

    * 适用于内存不规整、碎片较多的情况

#### 面试问答

**Q：对象分配内存有哪些方式？**

**A：**
对象分配内存主要有两种方式。
一种是指针碰撞，适用于内存规整的情况，只需要把空闲指针往后移动对象大小即可。
另一种是空闲列表，适用于内存不规整的情况，JVM 需要从空闲块列表中找到足够大的空间。

---

### 知识点 11：对象分配时的线程安全问题

多个线程同时创建对象时，可能会：

* 同时竞争同一块堆内存分配位置
* 导致内存分配冲突

JVM 常见处理方式：

* CAS
* TLAB

#### 面试问答

**Q：对象创建时为什么会有线程安全问题？**

**A：**
因为多个线程可能同时在堆中为对象分配内存，如果都去修改同一个内存分配位置，就可能发生冲突，所以对象分配过程需要保证线程安全。

**Q：JVM 如何解决对象分配时的线程安全问题？**

**A：**
JVM 常见的做法有 CAS 和 TLAB。
CAS 用于并发情况下安全地更新共享分配位置。
TLAB 则是给每个线程分配一小块线程本地分配缓冲区，优先在线程自己的区域里分配对象，从而减少竞争。

---

### 知识点 12：TLAB

TLAB 的全称是：

* Thread Local Allocation Buffer

本质：

* 堆中的一小块线程本地区域
* 不是独立内存区，仍然属于堆

作用：

* 减少多个线程在堆上分配对象时的竞争
* 提高对象分配效率

#### 面试问答

**Q：什么是 TLAB？**

**A：**
TLAB 是线程本地分配缓冲区。
JVM 会在堆中为每个线程预留一小块区域，线程创建小对象时优先在自己的 TLAB 中分配，从而减少多个线程同时分配对象时的竞争。

**Q：TLAB 有没有完全消灭 CAS？**

**A：**
没有。
TLAB 只能减少 CAS 竞争，不能完全消灭。
当对象太大或者 TLAB 空间不足时，仍然需要回到共享堆空间中分配，这时仍然可能使用 CAS。

---

### 知识点 13：零值初始化

在对象创建过程中，JVM 会先把对象占用的内存初始化为默认值，例如：

* `int` -> `0`
* `boolean` -> `false`
* 引用类型 -> `null`

#### 面试问答

**Q：为什么实例变量不手动赋值也能直接使用？**

**A：**
因为 JVM 在对象创建过程中会先对对象内存进行零值初始化，所以实例变量即使没有显式赋值，也会先拥有类型对应的默认值。

**Q：为什么对象创建时要先做零值初始化？**

**A：**
因为 JVM 需要保证对象的实例字段在构造器执行之前就处于确定的默认状态，这样对象的初始状态才是可预测的，实例变量也才能在没有显式赋值时拥有默认值。

---

### 知识点 14：对象头

对象头主要包含两部分：

* **Mark Word**
* **类型指针**

其中：

* Mark Word 中保存哈希码、GC 年龄、锁状态等运行时信息
* 类型指针指向对象所属类的元数据

#### 面试问答

**Q：对象头里包含哪些内容？**

**A：**
对象头主要包括 Mark Word 和类型指针。
Mark Word 中会保存对象运行时的一些状态信息，比如哈希码、GC 年龄、锁状态等。
类型指针则指向这个对象所属类的元数据。

---

### 知识点 15：对象头为什么保存锁信息

原因：

* Java 对象可以直接作为锁使用
* JVM 执行 `synchronized` 时，要根据对象头判断锁状态

#### 面试问答

**Q：对象头为什么要保存锁信息？**

**A：**
因为 Java 中对象可以直接作为锁使用，比如 `synchronized(obj)`。
JVM 在执行加锁操作时，需要根据对象头中的锁标志位判断当前对象的锁状态，所以对象头中必须保存锁相关信息。

---

### 知识点 16：`<init>` 的作用

对象在 `<init>` 执行前：

* 已经有了内存
* 已经有了默认零值
* 已经有了对象头

但还没有完成：

* 程序员定义的初始化逻辑

#### 面试问答

**Q：对象在 `<init>` 执行前就算完全可用了吗？**

**A：**
不算。
在 `<init>` 执行前，对象只是完成了内存分配、零值初始化和对象头设置，但程序员自定义的初始化逻辑还没有执行。
只有 `<init>` 执行完成后，对象才算真正按照程序员意图初始化完成。

## 四、LeetCode 78：子集

### 知识点 17：子集问题的本质

子集问题本质上是在做：

* 对每个元素进行“选”或“不选”的决策
* 枚举所有可能组合

例如 `[1,2]` 的子集有：

* `[]`
* `[1]`
* `[2]`
* `[1,2]`

#### 面试问答

**Q：什么是子集问题？**

**A：**
子集问题就是从原数组中选出任意个元素组成集合，包括一个都不选的空集。
它本质上是在枚举每个元素选或不选的所有可能组合。

---

### 知识点 18：为什么这题用回溯

因为它本质上是一棵决策树：

* 每个元素都面临选或不选两种分支
* 需要遍历所有分支

回溯适合这种：

* 枚举所有可能
* 走一条路，再退回来试另一条路

#### 面试问答

**Q：LeetCode 78 为什么适合用回溯？**

**A：**
因为子集问题本质上是在枚举每个元素选或不选的所有可能组合，可以看成一棵决策树。
回溯正适合用来遍历这种决策树，逐层做选择、递归深入、再撤销选择。

---

### 知识点 19：回溯的核心动作

回溯的核心就是三步：

1. 做选择
2. 递归
3. 撤销选择

#### 面试问答

**Q：回溯的核心思想是什么？**

**A：**
回溯的核心思想就是做选择、递归深入、再撤销选择恢复现场。
本质上就是试一条路，走完之后再退回来试另一条路。

---

### 知识点 20：`path` 和 `result` 的含义

在子集题中：

* `path`：当前递归路径，也就是当前正在构造的子集
* `result`：存放所有子集结果

#### 面试问答

**Q：子集题里 `path` 和 `result` 分别表示什么？**

**A：**
`path` 表示当前递归路径，也就是当前正在构造的子集。
`result` 用来保存所有已经收集到的子集结果。

---

### 知识点 21：为什么一进入递归就收集结果

因为在这题里：

* 当前 `path` 本身就是一个合法子集
* 不只是叶子节点才是答案

例如：

* `[]`
* `[1]`
* `[1,2]`
* `[1,2,3]`

这些在递归过程中都可能成为答案

#### 面试问答

**Q：为什么一进入递归就要把 `path` 加入 `result`？**

**A：**
因为在子集问题中，每一个递归节点上的当前路径本身就是一个合法子集，不只是递归结束时才算答案。
所以每进入一层递归，都要先把当前 `path` 收集起来。

---

### 知识点 22：为什么不能直接 `result.add(path)`

因为：

* `path` 是同一个会不断变化的列表
* 后续回溯会继续修改它
* 如果直接加入 `result`，之前保存的结果也会被改掉

正确做法：

* `result.add(new ArrayList<>(path))`

#### 面试问答

**Q：为什么不能直接写 `result.add(path)`？**

**A：**
因为 `path` 是同一个会不断变化的列表，如果直接把它放进 `result`，后面回溯过程对 `path` 的修改会影响之前已经保存好的结果。
所以必须使用 `new ArrayList<>(path)` 拷贝出当前路径的快照。

**Q：`new ArrayList<>(path)` 的本质作用是什么？**

**A：**
它的本质作用是把当前 `path` 的内容复制成一个快照保存下来，这样后面即使 `path` 继续变化，也不会影响已经保存好的结果。

---

### 知识点 23：为什么递归传 `i + 1`

原因：

* 当前层真正选中的元素位置是 `i`
* 下一层应该从当前选中元素的后面继续选
* 所以传 `i + 1`

而不是 `startIndex + 1`，因为：

* `startIndex` 是这一层开始遍历的位置
* 它在当前层循环中不递增
* 真正递增的是 `i`

#### 面试问答

**Q：为什么递归传的是 `i + 1`，不是 `startIndex + 1`？**

**A：**
因为当前层真正选中的元素位置是 `i`，所以下一层递归应该从当前选中元素的后面开始继续枚举，也就是从 `i + 1` 开始。
`startIndex` 只是这一层开始遍历的位置，在当前层循环中并不会随着枚举一起递增，所以不能用 `startIndex + 1`。

---

### 知识点 24：为什么要 `removeLast()`

`removeLast()` 的作用不是“随便删一个数”，而是：

* 撤销刚才的选择
* 恢复进入这一层之前的现场
* 让后续分支不受污染

#### 面试问答

**Q：为什么回溯时要删除最后一个元素？**

**A：**
因为 `path` 表示的是当前路径。
当前分支走完后，必须把刚刚加入的元素删掉，恢复到进入这一层之前的状态，这样才能继续尝试下一条分支。
如果不删除，后续路径状态就会被污染。

---

### 知识点 25：子集题的回溯模板

核心流程：

1. 收集当前结果
2. 枚举当前层可以选的元素
3. 做选择
4. 递归
5. 撤销选择

#### 面试问答

**Q：请你口述一下子集题的回溯思路。**

**A：**
这道题用回溯来做，因为子集问题本质上是在枚举每个元素选或不选的所有可能组合。
我用 `path` 表示当前路径，用 `result` 存放所有结果。
由于每一个递归节点上的当前路径本身就是一个合法子集，所以每进入一层递归都先把当前路径加入结果。
但是不能直接加入 `path`，因为它后面还会变化，所以要保存当前路径的快照。
然后从 `startIndex` 开始枚举后续元素，选中一个元素后递归处理下一层，下一层从 `i + 1` 开始，最后再撤销选择。

## 五、今天的高频易错点

### 知识点 26：JVM 易错点

* 元空间不是和方法区并列的概念
* 运行时常量池属于方法区
* 对象创建第一步是类加载检查，不是看有没有相同对象
* 对象头 = Mark Word + 类型指针
* 程序计数器不会 OOM 不是因为“小”，而是因为规范规定

#### 面试问答

**Q：对象头等于 Mark Word 吗？**

**A：**
不等于。
对象头主要由 Mark Word 和类型指针组成。

**Q：程序计数器为什么不会 OOM？是不是因为它占用空间小？**

**A：**
更准确的说法不是因为它小，而是因为 Java 虚拟机规范明确规定程序计数器不会发生 OutOfMemoryError。

---

### 知识点 27：子集题易错点

* 忘记收集当前路径
* 直接 `result.add(path)`
* 递归参数写成 `startIndex + 1`
* 忘记 `removeLast()`

#### 面试问答

**Q：子集题最容易写错的地方有哪些？**

**A：**
最容易写错的地方有四个。
第一，忘记一进入递归就收集当前路径。
第二，直接写 `result.add(path)`，没有保存快照。
第三，递归时把参数写成 `startIndex + 1`，而不是 `i + 1`。
第四，忘记在递归返回后执行撤销选择，也就是 `removeLast()`。

## 六、今日压缩版总复习

### 知识点 28：一句话总复习

* JVM：运行时数据区分为线程私有和线程共享；对象创建流程是
  **类加载检查 -> 分配内存 -> 零值初始化 -> 设置对象头 -> 执行 `<init>`**
* 子集：本质是枚举每个元素选或不选的所有可能；回溯核心是
  **做选择 -> 递归 -> 撤销选择**

#### 面试问答

**Q：请你用一句话概括今天 JVM 和子集两部分内容。**

**A：**
JVM 部分要掌握运行时数据区划分和对象创建流程；子集部分要掌握回溯模板，理解为什么要收集路径、保存快照、传 `i + 1` 以及撤销选择。

class Solution {
List<List<Integer>> result = new ArrayList<>(); // 行1
LinkedList<Integer> path = new LinkedList<>();   // 行2

    public List<List<Integer>> subsets(int[] nums) { // 行3
        backtracking(nums, 0); // 行4
        return result;          // 行5
    }

    // 回溯核心函数，我们逐行盯死
    private void backtracking(int[] nums, int startIndex) { // 行6
        result.add(new ArrayList<>(path));  // 行7 【就是这行把[1,2,3]加入了result】

        for (int i = startIndex; i < nums.length; i++) { // 行8 【循环开始】
            path.add(nums[i]);              // 行9 【加元素】
            backtracking(nums, i + 1);      // 行10【递归调用：进下一层】
            path.removeLast();              // 行11【撤销操作：你最关心的代码】
        } // 行12 【循环结束】
    } // 行13 【函数结束：回到上一层调用它的地方】
}
